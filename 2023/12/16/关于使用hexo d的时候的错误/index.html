关于使用hexo d的时候的错误

错误代码：

FATAL Something's wrong. Maybe you can find the solution here: https://hexo.io/docs/troubleshooting.html

解决方法（1）：

此方法对有些频繁操作GitHub，需要经常输入密码来验证的人也有帮助

  • 先配置配置user.name和user.email（配置过的不需要再配置了）

  • 查看SSH Key（网上有教程或者去我上一篇文章中找）

  • 生成密钥（网上有教程或者去我上一篇文章中找）

  • 复制密钥到github（网上有教程或者去我上一篇文章中找）

  • 验证是否配置成功（网上有教程或者去我上一篇文章中找）

  • 修改hexo的配置文件.config.yml：

    deploy:
    type: git
    repo: git@github.com:你的用户名/你的仓库名.git
    branch: master

解决方法（2）：

进入博客根目录，删除git提交文件夹，最后重新生成提交

解决方法（3）：

在文件浏览器中勾选显示隐藏的项目
进入我们博客目录的“.deploy_git.git”子目录，找到config文件
打开config文件，添加以下内容：

[user]
    email=你的邮箱
    name=GitHub用户名

虽然问题解决了，但是我还是有疑问，因为在Git Bash中我已经配置过邮箱了，测试也是生效了，HEXO这里就显示没有了。如果有大神希望可以给我解疑，谢谢
